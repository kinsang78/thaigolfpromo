# 작업 지시서: 일반 대화 메시지 필터링

## 배경
카카오 챗봇에 "안녕", "ㅋㅋ", "뭐야" 같은 비프로모션 메시지가 들어오면 Claude API가 파싱을 시도하다 실패하여 에러 메시지가 표시된다. 불필요한 API 호출 비용도 발생한다.

## 목표
1. 짧은 일반 대화는 API 호출 없이 바로 안내 메시지 반환 (비용 절약)
2. 애매한 메시지는 Claude가 판단하되, 프로모션이 아니면 빈 배열 반환
3. 빈 배열일 때 친절한 안내 메시지 표시

## 수정 대상
`api/parse-promotion.js`

## 변경 사항

### 1. 사전 필터링 함수 추가 (handler 함수 위에)

handler 함수 바로 위에 아래 함수를 추가한다:

```javascript
function isLikelyPromotion(text) {
  if (!text || text.trim().length < 10) return false;

  var keywords = [
    "골프", "그린피", "바트", "캐디", "카트", "프로모션", "할인",
    "올인", "라운드", "부킹", "예약", "티오프", "샷건",
    "CC", "GC", "골프장", "컨트리클럽",
    "THB", "IDR", "루피아",
    "주중", "주말", "오전", "오후",
    "포함", "별도", "불포함",
    "대회", "토너먼트", "참가비"
  ];

  var lower = text.toLowerCase();
  for (var i = 0; i < keywords.length; i++) {
    if (lower.indexOf(keywords[i].toLowerCase()) !== -1) return true;
  }

  // 가격 패턴 감지 (숫자 + 바트/원/루피아, 또는 콤마 포함 숫자)
  if (/\d{3,}/.test(text)) return true;

  return false;
}
```

### 2. handler 함수에 사전 필터링 적용

handler 함수에서 `if (!utterance)` 체크 바로 다음에 추가:

```javascript
  // 일반 대화 필터링
  if (!isLikelyPromotion(utterance)) {
    return res.json(kakaoResponse(
      "골프장 프로모션 정보를 보내주세요!\n\n" +
      "예시:\n" +
      "알파인 주중 2500바트 그린피+캐디+카트 포함 4/1~4/30"
    ));
  }
```

### 3. SYSTEM_PROMPT에 빈 배열 규칙 추가

handler 함수 내부의 SYSTEM_PROMPT 배열에서, "=== 기타 규칙 ===" 섹션의 기존 규칙 11번 다음에 12번을 추가:

```javascript
"12. 프로모션이 아닌 일반 대화(인사, 질문, 잡담 등)인 경우 빈 배열 []을 반환하세요.",
"    골프장명, 가격, 날짜 중 하나도 없으면 프로모션이 아닙니다.",
```

### 4. 빈 배열 처리 로직 추가

handler 함수에서 JSON 파싱 성공 후, `promotions = promotions.map(...)` 줄 바로 전에 추가:

```javascript
    // 빈 배열 처리 (프로모션이 아닌 메시지)
    if (promotions.length === 0) {
      return res.json(kakaoResponse(
        "프로모션 정보가 감지되지 않았습니다.\n\n" +
        "골프장명과 가격이 포함된 프로모션 메시지를 보내주세요.\n" +
        "예: 알파인 주중 2500바트 캐디카트포함 4/1~4/30"
      ));
    }
```

## 주의사항
- var 키워드 사용 유지
- 기존 코드 구조를 최대한 유지하고, 위 4가지만 삽입
- isLikelyPromotion의 키워드 목록에는 한글과 영문 모두 포함
- 숫자 3자리 이상이 있으면 가격일 가능성이 있으므로 통과시킴
- 안내 메시지에 예시를 포함하여 사용자가 올바른 형식을 알 수 있게 함

## 테스트 시나리오
1. "안녕" → 사전 필터링에서 걸림 → 안내 메시지 (API 호출 없음)
2. "ㅋㅋㅋ" → 사전 필터링에서 걸림 → 안내 메시지
3. "알파인 얼마야?" → "골프" 키워드 없지만 짧음. 하지만 "알파인"은 키워드에 없으므로 필터링됨. 이건 OK — 가격 정보가 없는 질문이므로.
4. "알파인 주중 2500바트 캐디카트포함" → "바트", "캐디", "카트" 키워드 매칭 → 통과 → 정상 파싱
5. "로얄방파인 4월 1일~30일 올인 2000바트" → "올인", "바트" 매칭 → 통과 → 정상 파싱
6. "오늘 날씨 어때?" → 10자 미만 아니지만 키워드 없고 숫자 없음 → 필터링 → 안내 메시지
