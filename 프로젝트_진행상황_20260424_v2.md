#### 진단 결과
51개 전부 Golfdigg에서 HTTP 200 응답하지만 `course:{name:...}` 구조조차 파싱 불가. 즉, 페이지가 존재하지 않거나 구조가 다름.

#### 근본 원인 (마틴 직접 확인)
**ALL_SLUGS 리스트가 구버전임.** Golfdigg가 슬러그를 변경했거나 일부 코스를 제거함:

| 기존 ALL_SLUGS | 실제 Golfdigg | 상태 |

| `river-dale-golf-club` | `riverdale-golf-club` | 하이픈 변경 |

| `thai-country-club` | (없음) | 제거됨 |
| `amata-spring-country-club` | (없음) | 제거됨 |

#### 슬러그 재수집 시도 (실패)
`slug-discovery.js`로 여러 리스팅 페이지를 크롤했으나:
- `/en/courses`, `/en/search?...` 등 대부분 404 (라우팅 경로 아님)
- `/en/hotdeals`는 200이지만 1개만 추출 (카드가 JS 동적 로딩)
- 홈페이지 `/en`에서 12개만 (프로모션 배너 링크만 정규식에 걸림)

**결론:** Golfdigg는 SPA이므로 HTML 단순 크롤로는 전체 코스 목록 확보 불가.

## 3. 🎯 다음 세션 할 일 — Golfdigg 전체 코스 가격 재수집

### 문제 정의
**Golfdigg 사이트에서 태국 전체 골프장 가격을 빠짐없이 수집하는 방법 개발**

### 시도해볼 접근법 (우선순위 순)

#### 1. Next.js `__NEXT_DATA__` 파싱
- Next.js 앱은 홈페이지 HTML 하단에 `<script id="__NEXT_DATA__" type="application/json">` 블록으로 전체 코스 데이터 임베딩
- 홈페이지 HTML 다운로드 후 이 JSON만 추출하면 전체 리스트 획득 가능

```js
const match = html.match(/<script id="__NEXT_DATA__"[^>]*>([^<]+)<\/script>/);
const data = JSON.parse(match[1]);
// data.props.pageProps에 대부분의 데이터 있음
```

#### 2. 백엔드 API 엔드포인트 직접 호출
- Golfdigg의 API URL 추정 (Next.js 관례상):
  - `https://golfdigg.com/api/courses?country=thailand`
  - `https://api.golfdigg.com/courses`
  - `https://golfdigg.com/_next/data/{BUILD_ID}/en.json` ← 빌드 ID 알면 가장 확실
- Chrome DevTools → Network 탭에서 실제 XHR 호출 확인 필요

#### 3. Google Site 검색 (차선책)
- `site:golfdigg.com/en/courses` 구글 검색으로 인덱싱된 코스 URL 수집
- 완벽하지 않지만 보완적 방법

#### 4. 모바일 앱 API 역공학
- iOS/Android 앱이 사용하는 백엔드 API가 웹보다 깨끗할 수 있음
- `golfdigg.com/api/v1/...` 같은 패턴 탐색
- 리스크: 시간 많이 듬

### 추천 시작 방법
**Chrome DevTools 열어서 Golfdigg 사이트 네트워크 탭 확인** → 실제 호출되는 API 엔드포인트 파악 → 그 API를 스크립트에서 호출
