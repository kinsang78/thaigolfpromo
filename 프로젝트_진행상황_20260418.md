## 1. 프로젝트 개요

### 서비스명
**타이인니 골프 프로모션** (GitHub: `kinsang78/thaigolfpromo`)

### 목적
태국·인도네시아 골프장의 에이전시 프로모션 가격을 카카오 챗봇으로 수집 → Claude API 파싱 → Supabase 저장 → 웹앱 조회

### 기술 스택
- **호스팅:** Vercel (스킬서버 + 웹앱)
- **DB:** Supabase (PostgreSQL) — `https://lkdhclaoxqgrlrjmsglm.supabase.co`
- **AI 파싱:** Anthropic Claude API (claude-haiku-4-5-20251001)
- **챗봇:** 카카오 i 오픈빌더 + 카카오톡 채널 "타이골프프로모션"
- **프론트:** 단일 index.html (바닐라 JS)
- **일반가격 수집:** 로컬 PC에서 Node.js 스크립트 실행

### 작업 방식
- **Opus** (claude.ai 채팅): 아이디어 논의, 설계, 결과물 검토
- **Claude Code** (Sonnet): 실제 코드 작성/수정. 작업 지시서(.md)를 프로젝트 폴더에 넣고 "파일을 읽고 그대로 수행해줘"로 전달
- **git push** → Vercel 자동 배포

### 개발 환경
- Windows PC, Claude Pro 구독
- Claude Code v2.1.100 설치 완료 (C:\thaigolfpromo에서 실행)

- .env 파일 설정 완료 (SUPABASE_URL, SUPABASE_SERVICE_KEY)

## 2. DB 구조 (현재)

### golf_courses 테이블
```sql
id SERIAL PRIMARY KEY,
name_ko TEXT,                    -- 한글명 (129개 전부 보유)
name_en TEXT UNIQUE,             -- 영문명
country TEXT NOT NULL,           -- '태국' 또는 '인도네시아'
region TEXT,                     -- 세부 지역
latitude DOUBLE PRECISION,      -- GPS 위도 (129개 전부 보유)
longitude DOUBLE PRECISION,     -- GPS 경도
golfdigg_slug TEXT,              -- Golfdigg URL 슬러그 (태국)
gogolf_id TEXT,                  -- GoGolf ID (인도네시아)
normal_weekday_am INTEGER,      -- 일반가 주중 오전
normal_weekday_pm INTEGER,      -- 일반가 주중 오후
normal_weekend_am INTEGER,      -- 일반가 주말 오전
normal_weekend_pm INTEGER,      -- 일반가 주말 오후
normal_price_currency TEXT,     -- 'THB' 또는 'IDR'
normal_price_includes TEXT,     -- 'green_fee_only' 또는 'green_fee_caddy_cart'
normal_price_source TEXT,       -- 'golfdigg' 또는 'gogolf'
normal_price_updated_at TIMESTAMPTZ,
created_at TIMESTAMPTZ,
updated_at TIMESTAMPTZ
```

### promotions 테이블
```sql
id UUID PRIMARY KEY,
golf_course TEXT NOT NULL,       -- 한글 골프장명

country TEXT,
region TEXT,
start_date DATE,
end_date DATE,
price_type TEXT,                 -- 'green_fee_only' | 'all_inclusive' | 'package'
green_fee JSONB,
currency TEXT,
includes_caddy BOOLEAN,
includes_cart BOOLEAN,
includes_hotel BOOLEAN,
includes_meal BOOLEAN,
caddy_fee INTEGER,
cart_fee INTEGER,
conditions TEXT,
contact_kakao TEXT,
contact_phone TEXT,
raw_message TEXT,
reported_by TEXT,
status TEXT,                     -- 'active'
latitude DOUBLE PRECISION,
longitude DOUBLE PRECISION,
created_at TIMESTAMPTZ,
updated_at TIMESTAMPTZ
```

## 5. 현재 이슈 (다음 대화에서 해결 필요)

### 이슈 1: 챗봇 저장 실패
**증상:** 카카오 챗봇에서 "등록 완료"라고 응답하지만 실제 DB에 저장되지 않음
**확인된 사례:** "로빈스우드 대회" 프로모션 — 챗봇은 성공 응답, DB에는 없음 (SELECT 결과 0건)
**의심 원인:** RLS 설정 후 service_role 키가 제대로 동작하지 않거나, Vercel 환경변수 문제
**확인 필요:**
1. Supabase에서 RLS 정책 확인: `SELECT * FROM pg_policies WHERE tablename = 'promotions';`
2. Vercel 환경변수에 SUPABASE_SERVICE_KEY가 정확히 설정되어 있는지 확인
3. Vercel Logs에서 "저장 에러" 메시지 확인

### 이슈 2: Golfdigg 전체 골프장 매칭 작업
**배경:** 가격 미보유 골프장의 Golfdigg slug를 하나씩 찍어보는 방식이 비효율적. Golfdigg 사이트의 slug가 예측과 다른 경우가 많음 (예: `legacy-golf-club` → 실제는 `legacy-golf-club-thailand`)
**발견한 Golfdigg API:**
- 슬롯/가격 API: `https://api.golfdigg.com/golfdigg/slot/v4/list/?courseId={ID}&date={YYYY-MM-DD}`
- 인증 불필요 (브라우저에서 직접 호출 가능)
- RSC 방식 대신 이 API를 사용하면 안정적
**확인된 courseId 매핑:**
- 블랙마운틴: `596c7cf4ef0a604a7de1b55e`
- 레이크우드: `575e976be4b0ea35e52e4ba9`
- 유니랜드: `5aabac17ef0acfa3ae364b4a`
- 레가시: `5965ccf1ef0a1f9d5657ef26`
- 에카차이: `5b5020e5ef0a07fa68ba2171`
- 탄야: `563c9e83e4b04e23d3d346c3`
**다음 할 일:** Golfdigg 전체 골프장 목록을 한 번에 가져오는 방법을 찾아서, 우리 DB의 golf_courses와 매칭 확인. Golfdigg 검색 API가 있을 수 있음 — 메인 페이지 Network 탭에서 확인 필요.

### 이슈 3: agency-scraper.js 업데이트

## 7. 핵심 파일 변경 이력

### parse-promotion.js (최신: 커밋 212a484)
- COURSE_MASTER 동적화: DB에서 golf_courses 전체 로딩 (loadCourseMaster 함수)
- SYSTEM_PROMPT: handler 내부에서 동적 생성, masterListText는 한글명 키만 포함
- correctWithMaster: COURSE_MASTER를 인자로 받음, golf_course_id 자동 설정
- isLikelyPromotion: 사전 필터링 (10자 미만 + 키워드 체크 + 숫자 3자리 패턴)
- 규칙 12번: 미등록 골프장도 가격 있으면 파싱하도록 수정
- insertNew/saveWithDedup: golf_course_id 저장

### index.html (변경 없음)
- golf_courses + promotions 분리 로딩 후 JS 머지
- 오전/오후 동일 가격 통합 표시
- 프로모션만 보기 토글
- 정렬: 이름순, 프로모션 우선, 가격, 마감임박, 가까운순
